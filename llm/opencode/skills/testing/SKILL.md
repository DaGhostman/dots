## What I do

- Recommend unit, integration, and end‑to‑end test strategies
- Assist in writing test code in multiple languages and testing frameworks
- Review test coverage and suggest improvements
- Promote Test-Driven Development (TDD) and behavior-driven testing patterns
- Guide on mocking, stubbing, and test isolation techniques
- Take note of performance degradations and/or tips

## When to use me

When you need to ensure your changes are safe, validate features, or build confidence in your codebase. Useful when starting a new test suite or improving existing test coverage.

## When NOT to use me

For debugging runtime issues—use Debugging skill. For performance testing—use Performance skill.
